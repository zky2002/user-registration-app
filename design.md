# 用户注册应用 - 设计文档

## 应用概述

一个简洁的用户注册应用，用户通过输入手机号和用户名完成注册，注册成功后进入欢迎页面。

---

## 屏幕列表

### 1. **注册屏幕** (Registration Screen)
- **路由**: `/` (主屏幕)
- **功能**: 用户输入手机号和用户名进行注册
- **内容**:
  - 应用标题/Logo
  - 手机号输入框（格式验证）
  - 用户名输入框
  - 注册按钮
  - 加载状态反馈

### 2. **成功屏幕** (Success Screen)
- **路由**: `/success`
- **功能**: 显示注册成功信息，展示用户信息
- **内容**:
  - 成功图标/动画
  - 欢迎文本
  - 用户信息展示（手机号、用户名）
  - 返回按钮或继续按钮

---

## 主要用户流程

### 流程 1: 用户注册
1. 用户打开应用，看到注册屏幕
2. 用户输入有效的手机号（11位数字）
3. 用户输入用户名（非空）
4. 用户点击"注册"按钮
5. 应用验证输入数据
6. 应用向后端发送注册请求
7. 后端返回成功响应
8. 应用导航到成功屏幕
9. 成功屏幕显示用户信息

### 流程 2: 返回注册
1. 用户在成功屏幕点击"返回"按钮
2. 应用导航回注册屏幕
3. 表单清空，用户可重新注册

## 颜色方案

| 元素 | 颜色 | 用途 |
|------|------|------|
| **主色** | `#007AFF` | 按钮、链接、强调 |
| **背景** | `#FFFFFF` (浅) / `#151718` (深) | 页面背景 |
| **文本主色** | `#11181C` (浅) / `#ECEDEE` (深) | 标题、正文 |
| **文本次色** | `#687076` (浅) / `#9BA1A6` (深) | 辅助文本、提示 |
| **成功色** | `#34C759` | 成功提示 |
| **错误色** | `#FF3B30` | 错误提示 |
| **输入框背景** | `#F2F2F7` (浅) / `#2C2C2E` (深) | 输入框背景 |

---

## 关键交互

- **输入验证**: 实时验证手机号格式和用户名长度
- **加载状态**: 注册时显示加载指示器
- **错误处理**: 显示清晰的错误提示信息
- **触觉反馈**: 按钮按下时提供触觉反馈（iOS）
- **安全区域**: 遵守 iOS 安全区域，适配刘海屏

---

## 技术栈

- **框架**: Expo + React Native
- **路由**: Expo Router
- **状态管理**: React Hooks
- **后端**: Node.js + Express (tRPC)
- **数据库**: MySQL/TiDB
- **认证**: Manus OAuth

---

## 开发优先级

1. **P0**: 注册屏幕 UI + 表单验证
2. **P1**: 后端 API 集成 + 注册逻辑
3. **P2**: 成功屏幕 UI + 页面导航
4. **P3**: 错误处理 + 加载状态
5. **P4**: 深色模式支持 + 触觉反馈
